GIAIC Sunday Afternoon Kubernetes Class

This Sunday ,we had a highly informative Kubernetes (K8s) session at GIAIC, led by Sir Ali Jawwad. The class focused on understanding why Kubernetes is essentialin todays cloud-native and containerized world, especially when applications need to scale beyond local machine limits.

Why Kubernetes?

We discussed the real-world problem of scaling applications:

Local machines have limited resources (8GB 16GB 32GB RAM)

As user traffic grows, containers alone are not enough

Cloud providers like AWS, Azure, and Google Cloud** use Kubernetes to handle:

Automatic scaling
Self-healing when containers fail
Load balancing across multiple services

Kubernetes acts as an orchestration layer that intelligently manages these challenges.

Core Concepts Covered

* Pods:
Containers do not run directly in Kubernetes they live inside Pods, which are the smallest deployable units in K8s.
In most real-world cases, one Pod contains a single container.
* kubectl:
The command-line tool used to interact with and manage Kubernetes clusters.
* Architecture Flow:
Image Container Pod Node**

Key Components:

* kubelet: Runs on each node and follows instructions from the Kubernetes control plane
* containerd: Container runtime responsible for managing container lifecycle

Key Takeaway

This session helped build a strong foundation of how Kubernetes works internally and why it is a critical technology for modern DevOps and cloud infrastructure.

Huge thanks to Sir Ali Jawwad for breaking down complex Kubernetes concepts in a clear and practical way. Grateful to GIAIC for providing such industry-relevant learning opportunities. Looking forward to more hands-on Kubernetes sessions ahead!
